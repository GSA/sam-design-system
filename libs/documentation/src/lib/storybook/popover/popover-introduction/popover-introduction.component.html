17.0.7 Migration

With version 17.0.7 of SAM Design System the popover was updated to match the latest design mockups and to address a number of long-standing issues that had been raised with the existing popover.

Changes

Positioning

With this update popovers can now reposition if their content will spill out of view. The position input can now take multiple direction tokens as input. If the first token in the list would place the popover off the user's screen, the next token in the list will be tested. The first token in the list that does not overflow will be the token that is used. If all of the provided tokens would result in the popover going off the screen, the first token in the list will be used. Take a look at the position demo for this dynamic positioning in action.

Breaking Changes

Close Behavior

Previously the close behavior was controlled by two inputs closeOnContentClick and closeOnClickOutside. Going forward the close behavior of the popover will be controlled by one input autoClose. The default behavior of the popover will remain the same, an update will only be required if either closeOnContentClick or closeOnClickOutside were being set on a popover. Below is a chart to guide in maintaining the same close behavior. The bolded values are the default value of these fields.

autoClose closeOnContentClick closeOnClickOutside
'false' false false
'inside' true false
'outside' false true
'true' true true

HTML Embedding

The way that HTML elements are passed to the popover are being update with this updated. Previously a tag such as #popover-content would have been applied to the element to be embedded, this tag would then be passed into the sdsPopover input. Going forward the HTML element to be embedded will need to be wrapped in an ng-template element which is tagged with the original popover-tag. See below for an example of the changes needed.

Pre-17.0.7
<p #tipContent>Hello, <b>World</b>!</p>
<button [sdsPopover]="tipContent" [position]="'bottom'" class="usa-button usa-button--base"></button>
Post-17.0.7
<ng-template #tipContent>
 <p>Hello, <b>World</b>!</p>
</ng-template>
<button [sdsPopover]="tipContent" [position]="'bottom'" class="usa-button usa-button--base"></button>
