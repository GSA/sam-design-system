Tooltip allows additional information to be displayed when a user "focuses" on an element that this directive is applied to.

17.0.7 Migration

With version 17.0.7 of SAM Design System the tooltip was updated to match the latest design mockups and to address a number of long-standing issues that had been raised with the existing tooltip.

Changes

Positioning

With this update tooltips can now reposition if their content will spill out of view. The position input can now take multiple direction tokens as input. If the first token in the list would place the tooltip off the user's screen, the next token in the list will be tested. The first token in the list that does not overflow will be the token that is used. If all of the provided tokens would result in the tooltip going off the screen, the first token in the list will be used. Take a look at the position demo for this dynamic positioning in action.

Breaking Changes

HTML Embedding

The way that HTML elements are passed to the tooltip are being update with this updated. Previously a tag such as #tooltip-content would have been applied to the element to be embedded, this tag would then be passed into the sdsTooltip input. Going forward the HTML element to be embedded will need to be wrapped in an ng-template element which is tagged with the original tooltip-tag. See below for an example of the changes needed.

Pre-17.0.7
<p #tipContent>Hello, <b>World</b>!</p>
<button [sdsTooltip]="tipContent" [position]="'bottom'" class="usa-button usa-button--base"></button>
Post-17.0.7
<ng-template #tipContent>
 <p>Hello, <b>World</b>!</p>
</ng-template>
<button [sdsTooltip]="tipContent" [position]="'bottom'" class="usa-button usa-button--base"></button>
